## C4C2C — Per-Unit Combat Authority

- เปลี่ยน worker kernel จาก aggregate power เป็น authoritative actor รายตัว: Hero, Monster, Tower, Garrison และ Core
- ทุก actor ใช้ immutable `maxHealth`, `armor`, `attackDamage`, `attackCooldownMs`, range, position และ deterministic actor ID
- server materialize `army_items` ตอน validate loadout และ `defenseUnits` ตอน deploy town จึงไม่ต้อง query content รายตัวใน worker (เลี่ยง N+1 และ scale ได้ง่าย)
- เพิ่มขีดจำกัด 50 attacker units, 201 defender units, 25,000 commands, cooldown ขั้นต่ำ 100 ms และตรวจ unique actor ID
- kernel ตรวจ army count/power เทียบ authority และต้องมี Core authoritative เพียงหนึ่งตัว
- command stream เพิ่ม `DEFEATED` และ `ATTACK` ที่อ้าง actor/target จริง; simulation version เป็น `fixed-tick-c4c2c-v2`
- Content Catalog schema 2 ส่ง combat payload ของ Monster/Tower/Hero และใช้ content version `content-c4c2-v1`
- เพิ่ม migration `009_per_unit_combat_authority.sql`
- แก้ worker target snapshot ให้คืนเฉพาะ immutable layout ไม่ใช่ payload wrapper ทั้งก้อน
- เพิ่ม pre-stake gate: snapshot schema ต่ำกว่า 2 หรือไม่มี `defenseUnits` ถูกปฏิเสธด้วย `TARGET_COMBAT_SNAPSHOT_STALE`

## C4C2D — Lifecycle Polling + Verified Replay Persistence

- เพิ่ม migration `010_authoritative_raid_replays.sql`: replay เป็น immutable row แบบ one-result/one-raid
- trusted worker ส่ง simulation version, tick/count, command-stream hash และ commands พร้อมผล
- backend ตรวจ version, command whitelist, identity length/format, tick order, count limit 25,000, final `COMPLETE` และ SHA-256 ก่อนแตะ settlement
- command stream ที่ผิด hash ถูก reject ก่อน wallet/escrow เปลี่ยน
- `GET /v1/raids/{raidId}` แสดง replay readiness/version/hash และอนุญาตเฉพาะ attacker/เจ้าของเมือง
- `GET /v1/raids/{raidId}/replay` reconstruct input จาก immutable town/loadout snapshots ไม่เชื่อ input ซ้ำจาก worker
- Unity เพิ่ม `RaidLifecycleReplayController`: poll แบบ bounded exponential backoff, อ่าน public API เท่านั้น และเริ่ม presentation หลัง replay/lifecycle identity ตรงกัน
- local/simulated incoming-defense ถูก guard ไม่ให้ poll attacker lifecycle ผิดเส้นทาง
- presentation ตรวจ command hash ซ้ำฝั่ง Unity ก่อนเล่น
- PostgreSQL เก็บ JSONB stream เฉพาะ prototype; production จะใช้ object storage adapter แล้วเก็บ metadata/hash/index ใน PostgreSQL
