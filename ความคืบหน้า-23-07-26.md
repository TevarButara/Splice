## C4D1B — Backup/Restore Drill

สถานะ: **เสร็จและผ่าน automated drill**

ทำแล้ว:

- consistent PostgreSQL exported snapshot สำหรับ dump/fingerprint/replay inventory
- bundle private `0700`, ไม่ overwrite และไม่รับ symlink/path traversal
- restore เข้า database และ replay root ใหม่ด้วย single transaction
- ตรวจ ledger, account balance, town snapshot hash, raid escrow, ทุก table fingerprint และ object pointer/bytes
- เปิด restored replay ผ่าน ASP.NET API ได้จริง
- corrupt/missing blob fail-closed ก่อนสร้าง target
- snapshot exporter ไม่ทิ้ง connection ค้าง

ผล local fixture หลายรอบ: backup 0–1 วินาที, restore+verifyต่ำกว่า 1 วินาที. นี่ไม่ใช่ production RTO; production ยังต้องมี WAL/PITR, object versioning และ encrypted/signed off-site backup.
